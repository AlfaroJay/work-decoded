## 3. Codebase & security (Next.js)

`npx tsc --noEmit` passes clean. No secrets committed (`.env*` correctly gitignored; only placeholders in `.env.local.example` and `scripts/`). Error handling is generally good (server-side logging, generic client messages, HTML escaping in the `/r/[slug]` interstitial, fail-closed availability).

**Architectural note:** the production form (`public/intake-form.html`) posts directly to a hardcoded Zapier webhook, so `src/app/api/submit/route.ts` and `src/components/DateTimePicker.tsx` are effectively dead code.

### Critical
- **C1 — Public intake webhook, unprotected.** `public/intake-form.html` exposes the raw `hooks.zapier.com` catch URL client-side: no auth, validation, rate limit, or bot defense. Forged POSTs can trigger SMS/invoice/calendar (cost + spam). *Fix:* proxy submissions through a server endpoint with validation + rate limiting + Turnstile/hCaptcha; never expose the raw hook.
- **C2 — Airtable formula "escaping" in `validate-code` is ineffective.** `route.ts:51` does `\'`-style escaping, which Airtable formulas ignore. Today the strict allowlist `^[A-Z0-9-]{4,32}$` is the only real defense. *Fix:* rely on (and document) the allowlist; drop the false-confidence replace.

### High
- **H1 — Record ID = auth.** `/api/session`, `/api/intake`, `/api/feedback` return/modify full client records using only the Airtable `recXXX` ID as the credential, no expiry, no rate limit. Record IDs leak (emails, logs, Zap history). Exposes sensitive intake (harassment/discrimination/legal). *Fix:* signed, expiring tokens (HMAC) or real consultant auth; scope returned fields.
- **H2 — No rate limiting on any public endpoint.** `validate-code` (with `Access-Control-Allow-Origin: *`) is a brute-forceable code oracle that even returns `codeId`/`packageType`/counts; `availability` proxies Google Calendar unthrottled (quota DoS). *Fix:* per-IP limits; tighten CORS; return only `{valid, tier, remaining}`.
- **H3 — Open redirect** in the URL shortener (`/r/[slug]` + `/api/shorten`): stored destination only checked for `http(s)`, no host allowlist → phishing redirector if the key leaks. *Fix:* allowlist hosts (`meet.google.com`, `*.workdecodedhq.com`) at create + redirect time.

### Medium / Low
- **M1 — DST bug in `availability`** (`route.ts:57-58,86,94`): hardcoded `-04:00` offset → availability math off by one hour Nov–Mar. (The live form computes the offset correctly; the API route doesn't.) *Fix:* derive ET offset via `Intl.DateTimeFormat … longOffset`.
- **M2 — DST bug in dead `submit` route** (`:82-84`): naive `new Date()` parse + `timeZone` double-handling. Fix if revived.
- **M3 — `next.config.js:3` `generateBuildId: () => null`** — should return a string or be removed.
- **M4 — `validate-code` leaks package metadata** to anonymous callers (ties into H2).
- **L1 — No server-side email/phone format validation** anywhere (presence-only).
- **L2 — Dead code** (`/api/submit`, `DateTimePicker`) still routable; either remove or bring under validation/rate-limiting.
